1) Backup + Verify

Captures files + database, validates archive integrity, SQL sanity, and checksums before upload.

2) Upload Backends

Use local by default. Optional cloud via rclone or gog. Optional backend failures don’t block required local baseline.

3) Restore Confidence

Runs staged restore checks before swap and keeps rollback path. Emits RESTORE_SUMMARY markers for evidence.

FAQ

Is Google Drive required?
No. Local backend is the baseline default. gog and rclone are optional backends.

Can I run this for multiple sites?
Yes. Use profile files under config/sites/<slug>.env and switch with use_site_profile.sh.

How do I prove release readiness?
Run pre-release checks and generate launch + PR evidence bundles from scripts in the repo.
